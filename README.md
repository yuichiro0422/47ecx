## 概要

本アプリは、商品購入機能および在庫管理機能を備えたECサイトです。  
ユーザーは商品を閲覧・購入することができ、管理者は商品の追加・編集・削除や在庫数の管理を行うことができます。

バックエンドを中心としたWebアプリケーション開発の理解を目的として、CRUD処理や認証機能、データベース設計を含めて実装しました。  
また、AWS（EC2）上へのデプロイを行い、公開環境での動作を意識した構成としています。
<br>
<br>
<br>

## 作った目的

本アプリは学校の課題として制作したものですが、バックエンドの理解を深めることを目的に、自主的に機能追加や設計改善を行いました。

<br>
<br>

## 主な機能

### ユーザー機能

- ユーザー登録・ログイン機能
- 商品一覧表示
- 商品詳細表示
- カート機能
- 商品購入機能

### 管理者機能

- 商品の追加・編集・削除（CRUD）
- 在庫数の管理（増減機能）

  <br>

## 技術スタック

- フロントエンド：HTML / CSS / JavaScript
- バックエンド：PHP
- データベース：MariaDB
- Webサーバー：nginx
- インフラ：AWS（EC2）
  <br>
  <br>
  <br>

## インフラ構成

ユーザー → nginx（EC2） → PHPアプリケーション → MariaDB

AWS EC2上にアプリケーションをデプロイし、Webサーバーとしてnginxを使用しています。  
バックエンドはPHP、データベースはMariaDBで構成しており、HTTPS化も行っています。

## 工夫した点

- 管理画面では、ユーザーの操作性を意識し、各商品ごとに在庫編集・商品情報編集・削除を行えるボタンを配置し、別ページではなくモーダル上で操作できる設計としました。  
   しかし、データベースのデータを用いた処理を行う際にはページ更新が発生する構成としていたため、モーダル表示時に商品情報を取得しようとすると画面が再読み込みされ、モーダルが閉じてしまうという課題がありました。  
   そこで非同期通信を導入し、ページ全体を更新せずに必要なデータのみを取得・更新できるようにすることで、モーダルを維持したまま操作できるよう改善しました。
  <br>
  <br>
- AWS（EC2）上にデプロイし、ローカル環境だけでなく公開環境での動作確認を行うことで、実運用を意識した開発を行いました。
  <br>
  <br>
- データベース接続情報を `.env` ファイルに分離することで、機密情報をコードから切り離し、安全性と保守性を意識した構成に改善しました。
